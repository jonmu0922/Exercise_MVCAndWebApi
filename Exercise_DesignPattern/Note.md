# Design Pattern
### Design Pattern 分類

Creational pattern

> 處理「產生 object」。主要的目的有二，一是希望封裝 concrete class，二是想封裝「如何產生並結合 concrete class 的 instance」的過程。

* Abstract Factory (抽象工廠)
* Builder
* Factory Method (工廠方法) [done]
* Prototype
* Singleton (單例模式) [done]

Structural pattern

> 可以合成 class 跟 object 到更大的結構中。

* Adapter
* Bridge
* Composite
* Decorator (裝飾者模式) [done]
  > 早餐店 -> 吐司和配料
* Facade
* Flyweight
* Proxy

Behavior pattern
> 重點在 class 與 object 間的互動，以及各自的責任。

* Chain of Responsibility
* Command
* Interpreter
* Iterator
* Mediator
* Memento
* Observer (觀察者模式) [done] 
	> 報社 -> 訂報者
* State 	
* Strategy (策略模式) [done] 
    > 購物車
* Template Method
* Visitor
